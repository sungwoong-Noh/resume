# 이력서 타겟 맞춤 최적화

`data/resume.ts` 파일을 읽고 타겟 포지션에 맞게 최적화 방안을 제안하라.

## 분석 항목

### 1. 포지션 키워드 분석
- 현재 profile.title 기준으로 해당 포지션에서 중요한 키워드 목록 도출
- 이력서에 포함된 키워드 vs 누락된 키워드 비교
- ATS(Applicant Tracking System) 통과를 위한 키워드 보강안

### 2. 강조점 재배치
- 타겟 포지션에 가장 관련 높은 경험/성과를 상단에 배치하는 방안
- 덜 관련된 경험의 축소/제거 제안
- skillCategories의 순서와 레벨 적절성

### 3. 차별화 포인트
- 동일 연차 지원자 대비 차별화할 수 있는 경험 식별
- 현재 잘 드러나지 않는 강점 발굴
- 약점을 강점으로 프레이밍하는 방법

### 4. 스토리라인 점검
- profile.intro → coreValues → experiences 흐름이 일관된 스토리를 전달하는지
- "이 사람은 ___를 잘하는 개발자다"라는 인상이 명확한지
- 채용 담당자가 30초 내 파악할 수 있는 핵심 메시지

## 출력 형식

현재 상태 분석과 개선안을 구분하여 제시하라.
우선순위(높음/중간/낮음)를 매겨 어떤 것부터 수정해야 효과적인지 안내하라.
구체적인 수정 문구를 포함하라.
단, 실제 파일 수정은 하지 말고 제안만 하라. 사용자가 승인하면 그때 수정하라.
