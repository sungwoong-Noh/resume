# 이력서 고도화 파이프라인

이력서 데이터(`data/resume.ts`)를 체계적으로 고도화하는 전체 파이프라인을 실행한다.
각 단계는 순차적으로 진행하며, 단계별로 사용자 승인을 받은 후 다음 단계로 넘어간다.

## 파이프라인 흐름

```
[1. 진단] → [2. 구조] → [3. 첨삭] → [4. 일관성] → [5. 맞춤법] → [6. 최적화] → [7. 최종검수]
```

## 실행 규칙

### 공통 규칙
- 각 단계에서 `data/resume.ts`를 분석하고 **제안만** 출력한다
- 사용자가 승인("적용해", "ㅇㅇ", "좋아", "넘어가" 등)하면 해당 제안을 파일에 반영한다
- 사용자가 거부하거나 수정 요청하면 제안을 조정한다
- 사용자가 "스킵"하면 해당 단계를 건너뛴다
- 각 단계 완료 후 진행 상황을 표시한다

### 진행 상황 표시 형식
매 단계 시작 시 아래 형식으로 현재 위치를 표시한다:
```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
[2/7] 구조 점검
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

## 단계별 상세

### 1단계: 현황 진단
`.claude/commands/resume-diagnose.md`의 지침을 따른다.
- 완성도, 포지션 정합성, 강약점을 진단
- 이후 단계에서 집중할 개선 포인트를 도출
- 출력 마지막에 "진단 결과를 확인하셨으면 다음 단계로 넘어갑니다" 안내

### 2단계: 구조 점검
`.claude/commands/resume-structure.md`의 지침을 따른다.
- STAR 기법 준수, 정보 계층, 임팩트 순서 점검
- 구조 변경이 필요한 항목의 before/after 제시

### 3단계: 내용 첨삭
`.claude/commands/resume-edit.md`의 지침을 따른다.
- 수치 구체화, 중복 제거, 액션 동사 강화
- [현재 문구] → [개선 문구] 형태로 제안

### 4단계: 일관성 점검
`.claude/commands/resume-consistency.md`의 지침을 따른다.
- 용어 통일, 날짜/기간 검증, 문체 통일

### 5단계: 맞춤법 점검
`.claude/commands/resume-spelling.md`의 지침을 따른다.
- 한국어 맞춤법, 영문 표기, 문장 품질

### 6단계: 타겟 최적화
`.claude/commands/resume-optimize.md`의 지침을 따른다.
- 포지션 키워드, 차별화, 스토리라인

### 7단계: 최종 검수
`.claude/commands/resume-review.md`의 지침을 따른다.
- 30초 테스트, 체크리스트, 완성도 점수

## 시작

파이프라인을 시작한다. 1단계 현황 진단부터 실행하라.
