# 이력서 최종 검수

`data/resume.ts` 파일을 읽고 최종 검수를 수행하라.
이전 단계(진단/구조/첨삭/일관성/맞춤법/최적화)에서 반영된 내용을 전제로 전체적인 완성도를 점검한다.

## 점검 항목

### 1. 30초 테스트
- 채용 담당자가 30초 내에 파악할 수 있는 정보:
  - 이 사람은 누구인가?
  - 무엇을 잘하는가?
  - 왜 뽑아야 하는가?
- 위 3가지가 명확히 전달되는지 평가

### 2. 전체 흐름 점검
- intro → coreValues → achievements → experiences → skills 순서의 스토리 일관성
- 각 섹션이 서로 보강하는 관계인지, 모순되는 부분은 없는지
- 전체적인 톤과 인상

### 3. 최종 체크리스트
- [ ] TODO/플레이스홀더 완전 제거 여부
- [ ] 모든 링크(GitHub, LinkedIn) 유효성
- [ ] 연락처 정보 정확성
- [ ] 오타/맞춤법 최종 확인
- [ ] 수치/날짜 정확성
- [ ] 용어 통일 완료
- [ ] 불필요한 정보 제거 완료

### 4. 개선 효과 요약
- 진단 단계에서 발견된 주요 이슈 vs 현재 상태 비교
- 남은 개선 사항이 있다면 목록화

## 출력 형식

체크리스트 형태로 통과/미통과를 명시하라.
미통과 항목이 있으면 구체적인 수정 방안을 제시하라.
전체적인 이력서 완성도를 점수(10점 만점)로 평가하고 근거를 설명하라.
