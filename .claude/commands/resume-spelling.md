# 이력서 맞춤법 및 문체 점검

`data/resume.ts` 파일을 읽고 맞춤법과 문체를 점검하라.

## 점검 항목

### 1. 한국어 맞춤법
- 띄어쓰기 오류
- 맞춤법 오류
- 조사 사용 오류
- 외래어 표기법 (예: 인프라스트럭처 vs 인프라)

### 2. 영문 표기
- 기술 용어 공식 표기 준수 (예: GitHub, JavaScript, TypeScript, ElasticSearch vs Elasticsearch)
- 대소문자 일관성
- 약어 첫 등장 시 풀네임 필요 여부

### 3. 문장 품질
- 지나치게 긴 문장 분리
- 불필요한 수식어 제거
- 능동태 전환
- 전문 용어의 적절한 사용

### 4. 가독성
- bullet point 길이 균일성 (1~2줄 권장)
- 핵심 키워드가 문장 앞쪽에 배치되어 있는지
- 기술적 깊이와 비전문가 이해도 사이의 밸런스

## 출력 형식

발견된 오류를 [위치: 필드명] [현재] → [수정안] 형태로 제시하라.
오류 유형별로 그룹핑하라.
단, 실제 파일 수정은 하지 말고 제안만 하라. 사용자가 승인하면 그때 수정하라.
