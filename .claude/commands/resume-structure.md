# 이력서 구조 점검

`data/resume.ts` 파일을 읽고 구조적 관점에서 점검하라.

## 점검 항목

### 1. STAR 기법 준수 여부
각 프로젝트의 achievements를 STAR(Situation-Task-Action-Result) 관점에서 분석하라.
- 상황(S)과 과제(T)가 명시되어 있는지
- 행동(A)이 구체적인지
- 결과(R)가 정량적인지
- 부족한 요소가 있으면 개선안을 제시하라

### 2. 정보 계층 구조
- highlights와 projects.achievements 간 역할 분담이 명확한지
- 상위 레벨(highlights)은 요약, 하위 레벨(projects)은 상세여야 함
- 중복 기술이 있으면 어디에 남기고 어디에서 제거할지 제안

### 3. 임팩트 순서 배치
- 각 회사 내 프로젝트 순서가 임팩트 기준으로 적절한지
- 가장 인상적인 성과가 먼저 보이는지
- 스캔 가독성 (30초 내 핵심 파악 가능성)

### 4. 섹션 간 밸런스
- experiences vs sideProjects vs certifications 비중이 적절한지
- 각 프로젝트별 achievements 개수가 균형잡혀 있는지

## 출력 형식

각 프로젝트별로 현재 구조와 개선안을 비교하여 제시하라.
변경이 필요한 항목은 구체적인 수정 문구를 포함하라.
단, 실제 파일 수정은 하지 말고 제안만 하라.
