# 이력서 현황 진단

`data/resume.ts` 파일을 읽고 다음 관점에서 현황을 진단하라.

## 점검 항목

### 1. 완성도 점검
- TODO나 플레이스홀더가 남아있는 섹션 식별
- 비어있는 필드 (techStack: [], 빈 문자열 등) 찾기
- 각 섹션별 내용 충실도 평가 (상/중/하)

### 2. 포지션 정합성
- `profile.title`에 명시된 포지션과 실제 경력 내용의 일치도
- 현 직장 포지션과의 갭 분석
- 핵심 역량이 타겟 포지션을 잘 어필하는지

### 3. 구조 분석
- 경력 기술 순서 적절성
- 성과 중복 여부 (동일 성과가 여러 곳에 반복되는지)
- 각 프로젝트별 성과 개수와 품질 밸런스

### 4. 강점/약점 요약
- 현재 이력서의 3가지 강점
- 개선이 필요한 3가지 약점
- 우선 개선 추천 순서

## 출력 형식

마크다운 테이블과 리스트를 활용하여 구조화된 진단 리포트를 작성하라.
각 항목에 심각도(높음/중간/낮음)를 표시하라.
