# 이력서 내용 첨삭

`data/resume.ts` 파일을 읽고 내용 품질을 개선하라.

## 첨삭 기준

### 1. 수치 구체화
- 모호한 표현("많은", "크게", "대폭" 등)을 정량 지표로 전환
- 이미 수치가 있는 항목도 맥락이 부족하면 보강 (예: "왜 그 수치가 의미있는지")
- 비교 기준이 명확한지 (before → after)

### 2. 중복 제거 및 차별화
- 동일 성과가 여러 위치에 반복되는 경우 각 위치별 역할에 맞게 차별화
  - intro: 임팩트 중심 한 줄 요약
  - highlights: 핵심 결과 + 방법론
  - projects.achievements: 구체적 기술/과정 상세
- 완전히 중복되는 문구는 하나로 통합

### 3. 액션 동사 강화
- 수동적 표현("~되었습니다", "~했습니다")을 능동적 표현으로 변환
- 시작 동사를 다양화 (설계/구축/주도/달성/최적화/도입/자동화 등)
- 각 bullet이 행동과 결과를 모두 포함하도록

### 4. 기여도 명확화
- 팀 프로젝트에서 "내가 한 것"이 명확히 드러나는지
- "팀 내 최초", "직접", "주도" 등 기여 범위를 나타내는 표현 점검
- 모호한 역할 기술 개선

### 5. 불필요한 정보 제거
- 타겟 포지션과 무관한 내용
- 너무 오래되어 임팩트가 약한 기술/경험
- 과도하게 상세한 구현 디테일

## 출력 형식

각 항목별로 [현재 문구] → [개선 문구] 형태로 제시하라.
변경 이유를 간단히 설명하라.
사용자가 승인하면 적용할 수 있도록 정확한 문구를 제공하라.
단, 실제 파일 수정은 하지 말고 제안만 하라. 사용자가 승인하면 그때 수정하라.
