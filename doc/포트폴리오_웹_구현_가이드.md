# 포트폴리오 웹 구현 가이드

> **URL**: `localhost:3000/portfolio` (개발) / 배포 시 `/portfolio`  
> **작성일**: 2026-04-14

---

## 1. 전체 구조

```
app/
└── portfolio/
    └── page.tsx                        ← 포트폴리오 라우트 진입점

components/
└── portfolio/
    ├── PortfolioHero.tsx               ← 커버 페이지 (이름 / 소개 / 키워드)
    ├── CareerOverview.tsx              ← Career Overview 타임라인
    ├── ProjectDetail.tsx               ← Project Detail 상세 섹션
    ├── PortfolioContact.tsx            ← 마지막 연락처 페이지
    └── PrintButton.tsx                 ← PDF 저장 버튼 (우하단 고정)

data/
└── portfolio.ts                        ← 포트폴리오 데이터 단일 관리 파일
```

**내용 수정은 `data/portfolio.ts` 하나만 수정하면 모든 섹션에 자동 반영됩니다.**

## 3. 데이터 구조 (`data/portfolio.ts`)

### 3-1. `portfolioProfile` — 커버 페이지 정보

```ts
portfolioProfile = {
  name: '노성웅',
  tagline: '"건강한 개발문화로..."',
  role: 'Back-end Engineer',
  email: '...',
  phone: '...',
  github: 'sungwoong-Noh',
  workingStyles: [{ icon, title, description }, ...],   // Core Working Style 3가지
  keywords: [{ tag, description }, ...],                // Keyword 카드 3가지
  introduction: ['단락1', '단락2', '단락3'],             // Introduction 본문
}
```

### 3-2. `portfolioProjects` — 프로젝트 배열 (7개)

각 프로젝트는 아래 타입을 따릅니다:

```ts
type PortfolioProject = {
  id: string                    // URL slug 용도 (고유값)
  name: string                  // 프로젝트명
  period: string                // 기간 (예: '2024.08 ~ 2025.07')

  // Career Overview 섹션에 표시
  overviewDescription: string   // 짧은 프로젝트 설명 (2-3문장)
  overviewContributions: string[] // 기여도 bullet (3-4개)

  // Project Detail 섹션에 표시
  detailParagraphs: string[]    // 상세 설명 단락 배열
  keyResponsibilities: string[] // Key Responsibilities bullet

  techStack: {
    frontend?: string[]
    backend?: string[]
    infra?: string[]
  }

  screenshotSrc?: string        // 스크린샷 이미지 경로 (선택, 없으면 그라디언트)
  gradientClass: string         // 스크린샷 없을 때 폴백 그라디언트 (Tailwind 클래스)
}
```

**프로젝트 순서** = 배열 순서가 Career Overview / Project Detail 양쪽 모두의 표시 순서입니다.

## 4. 스크린샷 이미지 추가 방법

### 4-1. 이미지 파일 위치

```
public/
└── portfolio/
    ├── ppp-cloud.png
    ├── commerce.png
    ├── ntops.png
    ├── rpms.png
    ├── science.png
    ├── sms.png
    └── kepco.png
```

`public/` 폴더에 넣으면 `/portfolio/파일명.png` 경로로 접근됩니다.

### 4-2. `data/portfolio.ts` 에서 경로 지정

```ts
// 예시: PPP Cloud 프로젝트
{
  id: 'ppp-cloud',
  ...
  screenshotSrc: '/portfolio/ppp-cloud.png',   // ← 이 줄 추가
  gradientClass: 'from-slate-700 via-slate-600 to-slate-800',  // 폴백용 유지
}
```

### 4-3. 이미지 권장 스펙

| 위치 | 권장 크기 | 비율 |
|------|----------|------|
| Career Overview 카드 배너 | 600 × 200 px | 3:1 |
| Project Detail 상단 배너 | 1200 × 400 px | 3:1 |

- 포맷: `.png` 또는 `.webp` (용량 최소화)
- 실제 서비스 화면 스크린샷 또는 목업 이미지 권장

## 5. 디자인 커스터마이징

### 5-1. 색상

| 역할 | 현재 값 | 위치 |
|------|---------|------|
| 주 강조색 (초록) | `#2a6041` | 모든 컴포넌트 직접 지정 |
| 배경 밝은 크림 | `#f5f0e8` | Hero, ProjectDetail 섹션 |
| 배경 진한 크림 | `#eee8da` | CareerOverview, Contact 섹션 |
| 카드 테두리 | `#d6e8dc` | 카드 border |
| 타임라인 선 | `#b5cfc0` | CareerOverview 세로선 |

색상 변경 시 각 컴포넌트 파일에서 해당 hex 코드를 일괄 교체하면 됩니다.

### 5-2. 폰트

- **본문**: Inter (`tailwind.config.js` → `font-sans`)
- **Career Overview / Project Detail 헤더**: Playfair Display (`font-serif` 클래스)

폰트 변경 시 `app/globals.css` 상단 Google Fonts import URL과 `tailwind.config.js`의 `fontFamily` 를 수정합니다.

### 5-3. 그라디언트 폴백 색상 변경

스크린샷을 아직 추가하지 않은 프로젝트는 `gradientClass` 값으로 색상이 표시됩니다.  
`data/portfolio.ts`의 각 프로젝트 `gradientClass` 필드에서 Tailwind gradient 클래스를 교체합니다.

```ts
// 예시
gradientClass: 'from-emerald-700 via-teal-600 to-cyan-700'
```

## 6. Career Overview 타임라인 구조

```
[좌우 교차 배치]
짝수 인덱스 (0, 2, 4, 6) → 왼쪽 카드
홀수 인덱스 (1, 3, 5)    → 오른쪽 카드

중앙 세로선: position absolute, top-0 ~ bottom-0
타임라인 점: 각 행 position absolute, top-5 (20px)
```

모바일(md 미만)에서는 세로선·점이 숨겨지고 카드가 1열로 표시됩니다.

## 7. PDF 다운로드

- 우하단 고정 버튼 `PrintButton` 클릭 → `window.print()` 실행
- 브라우저 인쇄 창에서 **"PDF로 저장"** 선택
- 인쇄 시 버튼은 자동으로 숨겨집니다 (`print:hidden`)

### 인쇄 품질 팁

- 브라우저 인쇄 설정에서 **배경 그래픽 인쇄** 옵션을 켜야 배경색·그라디언트가 출력됩니다
- 여백: "없음" 또는 "최소" 권장
- 용지 방향: 세로

## 8. 프로젝트 추가/삭제

### 추가

`data/portfolio.ts`의 `portfolioProjects` 배열에 항목을 추가하면 Career Overview와 Project Detail 양쪽에 자동으로 표시됩니다.

### 삭제

배열에서 해당 객체를 제거하면 됩니다.

### 순서 변경

배열 순서를 바꾸면 표시 순서가 바뀝니다.
